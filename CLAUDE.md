DATABASE & POSTGREST RELATIONSHIP CONVENTIONSAmbiguous Relationship Join Hints: Because multiple tables share dual Foreign Key references (e.g., users and departments), PostgREST queries will crash with an ambiguous relationship error. Always supply an explicit hint naming the foreign key column:// Example Pattern:
const { data } = await supabase.from('users').select('id, departments!department_id ( id, name )');
Self-Joins: Direct database-level self-joins (like users referencing users!manager_id) produce ambiguous PostgREST responses. Resolve this by executing two distinct queries and generating an in-memory matching index dictionary (managerMap).Naming Rules: Database tables/columns use snake_case (plural), utility files use kebab-case, React components use PascalCase.5. FRONTEND & CANVAS RULES (REACT FLOW & SHADCN)A. React Flow & Zustand GuidelinesKeep nodes in Zustand canvas-store.ts defined as a strict type: Node<NodeData>[]. Cast explicitly at call-sites during serialization.Define nodeTypes completely outside the main FlowCanvas component to prevent visual components from flickering during reactive state loops.Auto-Save Filter: React Flow triggers onNodesChange on simple actions like element selection and dimension shifts. Filter updates strictly to prevent excessive database writes:const SAVE_NODE_CHANGES = new Set(['add', 'remove', 'position', 'reset']);
If isReadOnly is active, suppress all triggerSave() cycles and freeze canvas interactivity.B. Radix/Shadcn Element SafeguardNever supply an empty string value value="" inside Radix Select Items (e.g., standard dropdown clear options). This causes an immediate component-tree crash.Fix: Use a placeholder string like value="all", and update query filter parameters to treat "all" as a cleared/disabled filter.C. Build-Safe Production HygieneUnused declarations or imports trigger compilation failures in production.Fix: Prefix any intentionally unused properties with an underscore * or apply an explicit eslint-disable wrapper if resolving a complex hook loop.6.
